Что сделано по безопасности

Выполнены самые важные изменения для защиты админки, вопросов, результатов тестов, оплаты, контента и базы данных.

  1. Административный доступ.

    Вместо общего пароля из базы теперь используется отдельный аккаунт администратора с e-mail, паролем и серверной проверкой прав.

    Нельзя открыть админку простой подменой данных в браузере или прочитать общий пароль из базы.

  2. Доступ к базе данных.

    Для посетителей и администратора установлены разные, строго ограниченные права доступа.

    Посетитель не может менять тесты, вопросы, настройки и чужие результаты.

  3. Правильные ответы и подсчёт результата.

    Публичная часть сайта получает вопросы без правильных ответов. Подсчёт выполняется на сервере.

    Нельзя заранее скачать ответы или подделать результат в браузере.

  4. Доступ к результатам.

    Для каждой тестовой сессии создаётся отдельный случайный токен доступа.

    Одного ID сессии недостаточно, чтобы прочитать чужой результат, e-mail или ответы.

  5. Оплата.

    При создании оплаты проверяется и ID сессии, и её секретный токен.

    Нельзя связать оплату с чужим результатом без доступа к этой сессии.

  6. Отправка результатов по e-mail.

    Отправка писем доступна только доверенной серверной части приложения.

    Посторонний не сможет использовать её для рассылки или получения чужих данных.

  7. Форматирование контента.

    Произвольный HTML больше не исполняется. Поддерживается безопасное форматирование: текст, абзацы, списки, разделители и жирное выделение.

    Скрипты и опасные обработчики событий не запускаются.

  8. Режим совместимости с прежним контентом.

    Если старый контент будет отображаться неправильно, можно временно включить ограниченную совместимость с HTML.

    В этом режиме опасные теги и атрибуты всё равно удаляются.

  9. Изображения вопросов.

    Просмотр изображений оставлен публичным, а загрузка, замена и удаление доступны только администратору.

  10. Служебные данные.

    Из логов и аналитики убраны лишние e-mail и идентификаторы тестовых сессий.

  11. Защита от случайного возврата проблем.

    Проверки безопасности, безопасного форматирования, линтер и сборка запускаются при каждой публикации сайта.

Что изменилось для пользователей

Проходить тест по-прежнему можно без аккаунта. Вопросы, оплата и результаты работают по прежнему сценарию.

Дополнительная защита работает автоматически: браузер сохраняет токен текущей сессии и удаляет его из адресной строки после открытия результата.

Из заметных изменений только одно: для административной панели теперь нужен настоящий аккаунт администратора с e-mail и паролем.
